# MarioTrickster — 经典关卡局部挑战 ASCII 白盒模板集

本文档基于对 Celeste、Mega Man、VVVVVV、Super Mario Bros、Super Meat Boy 等经典 2D 平台跳跃游戏的局部难点截图分析，提取了 **8 个精妙的挑战片段**，严格按照以下规则转化为可直接使用的 ASCII 模板。

---

## 物理法则与字典速查

| 约束 | 数值 | 说明 |
|------|------|------|
| 水平安全跨越 | 4 格 | 普通玩家可轻松跨越 |
| 水平极限跨越 | 5 格 | 含 coyote time，需精确操作 |
| 垂直安全跳高 | 2 格 | 普通玩家可轻松达到 |
| 垂直极限跳高 | 2.5 格 | 原地起跳绝对上限（PhysicsMetrics.MAX_JUMP_HEIGHT） |
| 模板宽度范围 | 20 ~ 30 字符 | |
| 模板高度范围 | 10 ~ 15 行 | |
| 空白填充字符 | `.`（英文点号） | 绝对禁止空格 |

| 字符 | 含义 | 字符 | 含义 |
|------|------|------|------|
| `#` | 地面/实心砖 | `=` | 平台 |
| `-` | 单向平台 | `W` | 墙壁 |
| `^` | 地刺 | `~` | 火焰 |
| `P` | 摆锤 | `B` | 弹跳平台 |
| `C` | 崩塌平台 | `>` | 移动平台 |
| `E` | 弹跳怪 | `e` | 简单敌人 |
| `F` | 伪装墙 | `H` | 隐藏通道 |
| `o` | 金币 | `M` | 起点 |
| `G` | 终点 | | |

> **复制方法**：每个模板都包裹在代码块中。在 GitHub / Markdown 预览器中，点击代码块右上角的复制按钮即可。如果手动选择，请从第一个字符选到最后一个字符，**不要多选代码块外的空行**。粘贴到 Unity 的 Custom Template Editor 文本框后，点击 "Build From Text" 即可生成。

## 模板 1：地刺深渊与诱饵金币 (Spike Abyss & Bait)

**灵感来源**：Celeste 早期关卡的开放空间地刺挑战。

**设计理念**：在深渊上方悬浮金币作为"诱饵"，引诱玩家冒险跳跃。两侧有安全平台可以落脚，但金币正下方是地刺，拿到金币后必须精确落在安全区。这是 GMTK "Introduce - Develop" 模式的典型应用——先让玩家看到简单的深渊跳跃，再用金币引诱他们尝试更危险的路线。

```
M........................
.........................
..=.................=..G.
.........................
.........................
.......o...o...o.........
.........................
....=...........=........
....^...........^........
.......=.....=...........
.......^.....^...........
#########################
```

**尺寸**：25 x 12

### 动态参数建议

金币 (`o`) 可以设置轻微的上下浮动动画（振幅 0.2 格，周期 2 秒），增加视觉吸引力。如果想提升难度，可以将第 8 行的两个 `=` 替换为崩塌平台 `C`，迫使玩家拿到金币后不能停留，必须立刻跳走。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **冲刺 (Dash)** | Celeste | 允许玩家在空中改变方向和速度，大幅扩展跳跃路线的可能性。没有冲刺时，跳跃轨迹是固定的抛物线，容错率更低 |
| **抓墙/滑墙 (Wall Cling)** | Celeste, Mega Man X | 碰到墙壁可以短暂停留并蹬墙跳，开辟垂直方向的设计空间 |

## 模板 2：Z 字攀爬塔 (Zigzag Climb Tower)

**灵感来源**：Mega Man 经典的垂直攀爬关卡。

**设计理念**：将关卡从水平方向旋转为垂直方向。左右交错的平台形成"Z"字形上升路径，每个平台边缘放置巡逻敌人。玩家必须在观察敌人巡逻节奏的同时，精确计算起跳时机。这是一种"节奏型挑战"——不是反应速度的考验，而是耐心观察和时机把握的考验。

```
....................
.G..................
.=..................
....................
..............=.....
..............e.....
....................
.....=..............
.....e..............
....................
..............=.....
..............e.....
....................
.....=..............
.M..................
####################
```

**尺寸**：20 x 16

### 动态参数建议

简单敌人 (`e`) 建议设置为在所在平台上来回巡逻（Patrol 模式），速度 1.5 格/秒，到达平台边缘后掉头。玩家需要在敌人走向远端时起跳，在敌人返回前安全落到平台上。如果想增加变化，可以让不同层的敌人速度略有差异（1.0 ~ 2.0 格/秒），打破玩家形成的固定节奏。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **梯子 (Ladder)** | Mega Man | 允许玩家在垂直方向自由移动并停留，可以在梯子上观察和射击。没有梯子时，垂直移动只能靠跳跃，节奏更紧凑 |
| **远程攻击 (Shooting)** | Mega Man, Contra | 可以在安全距离消灭敌人。没有远程攻击时，只能踩踏，风险更高 |

## 模板 3：移动平台与弹跳怪协奏 (Platform & Bouncer Duet)

**灵感来源**：综合了 Donkey Kong Country 的矿车关卡和 Mario 系列的移动平台设计。

**设计理念**：两个移动平台分别做水平和垂直运动，中间放置一个弹跳怪。玩家必须从左侧移动平台起跳，踩中弹跳怪获得额外高度，然后落到右侧移动平台上。这是一个"三段式跳跃"——每一段都依赖前一段的成功，形成连锁反应。金币引导线暗示了正确的跳跃路径。

```
..........................
..........................
..........................
....................G.....
...................==.....
..........................
..........o...o...........
..........E...............
..........................
.o....................o...
.>....................>...
M.........................
##########################
```

**尺寸**：26 x 13

### 动态参数建议

左侧移动平台 (`>`) 建议设置为水平往复运动（Ping-Pong 模式），移动范围 4 格，速度 1.0 格/秒，这样玩家有足够的时间站稳并观察弹跳怪的位置。右侧移动平台建议设置为垂直往复运动，移动范围 3 格，速度 1.5 格/秒，比左侧略快，增加落点预判的难度。弹跳怪 (`E`) 建议设置为原地跳跃（Jump 模式），跳跃高度 2 格，周期 1.5 秒。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **传送带 (Conveyor Belt)** | Mega Man, Mario | 改变玩家站在上面时的水平速度，增加跳跃落点预判的难度。顺向传送带加速、逆向减速，可以与移动平台组合出非常复杂的时序挑战 |
| **弹簧 (Spring)** | Sonic | 与弹跳怪类似但可控性更高，玩家主动踩上去获得弹射。弹跳怪是被动的、有风险的，弹簧是主动的、安全的 |

## 模板 4：火焰走廊与摆锤 (Fire Corridor & Pendulum)

**灵感来源**：Super Meat Boy 的高压迫感通道 + 城堡关卡的经典摆锤设计。

**设计理念**：狭窄的水平通道，上下都是火焰，中间有摆锤阻挡。通道高度仅 4 格，玩家的跳跃高度是 3 格，意味着跳跃时头顶几乎擦着上方的火焰。摆锤在通道中摆动，玩家必须在摆锤摆到一侧时快速通过。这是纯粹的"时间窗口"挑战。

```
##############################
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
..............................
..............................
.M....P.....P.....P.....G.....
..............................
..............................
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
##############################
```

**尺寸**：30 x 9

### 动态参数建议

摆锤 (`P`) 建议设置为从天花板悬挂，摆动角度 60 度，周期 2.5 秒。三个摆锤的初始相位应该错开（0 度、120 度、240 度），这样玩家不能一口气冲过去，必须在每个摆锤前停下来等待时机。火焰 (`~`) 如果支持动态效果，可以设置为周期性喷发（喷 2 秒，停 1 秒），与摆锤形成双重时序压力。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **旋转锯片 (Circular Saw)** | Super Meat Boy | 沿固定轨道做圆周运动的致命障碍。与摆锤的区别在于轨迹是可预测的圆形，而摆锤是来回摆动。锯片可以放在平台边缘做圆周运动，迫使玩家精确计时 |
| **激光 (Laser Beam)** | Mega Man | 周期性开关的水平/垂直光束。与火焰的区别在于激光是瞬间致死的线段，而火焰是持续伤害的区域 |

## 模板 5：崩塌平台逃亡 (Crumbling Escape)

**灵感来源**：Celeste 的崩塌平台追逐段落 + Indiana Jones 式的"地板塌陷"桥段。

**设计理念**：整个关卡的平台全部是崩塌平台 (`C`)，玩家踩上去后会在短暂延迟后消失。这意味着玩家**不能停下来**，必须持续向前跑跳。底部是地刺，掉下去就死。金币放在跳跃路径上，引导玩家走正确的路线。

```
M........................
.........................
CC...................CC.G
..o.................o....
....CC...........CC......
......o.........o........
........CC...CC..........
..........o.o............
...........B.............
..........^^^............
#########################
```

**尺寸**：25 x 11

### 动态参数建议

崩塌平台 (`C`) 建议设置踩踏后延迟 0.5 秒开始抖动，再过 0.3 秒后消失，3 秒后重生。这个时间窗口要求玩家在 0.8 秒内完成起跳。弹跳平台 (`B`) 放在最底部的地刺上方，作为"最后的救命稻草"——如果玩家从崩塌平台掉落，还有一次弹跳的机会回到上方。金币 (`o`) 形成一条从左上到右下再到右上的"V"字形引导线。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **追逐者 (Chaser)** | Celeste (Badeline), Rayman | 一个从起点开始追赶玩家的不可消灭的威胁。与崩塌平台结合，形成前有断路、后有追兵的极限压迫感 |
| **计时器 (Timer)** | 多种 | 全局倒计时，时间到则关卡失败。与崩塌平台结合，增加心理压力 |

## 模板 6：伪装墙密室与隐藏通道 (Secret Chamber)

**灵感来源**：Super Mario Bros 的隐藏区域 + 银河恶魔城的探索设计。

**设计理念**：表面上看是一条简单的直线通道，但墙壁中隐藏着伪装墙 (`F`) 和隐藏通道 (`H`)。好奇的玩家如果尝试撞墙，会发现一个隐藏的密室，里面有大量金币作为奖励。这种设计奖励探索行为，让玩家养成"到处试试"的习惯。

```
#########################
#.......#FFFFF#.........#
#.......#.ooo.#.........#
#.......#.ooo.#.........#
#.......H.ooo.H.........#
#.......#.....#.........#
#.......#######.........#
#.......................#
#.......................#
#..e.........e..........#
#M...............=...=.G#
#########################
```

**尺寸**：25 x 12

### 动态参数建议

伪装墙 (`F`) 在视觉上应该与普通墙壁 (`#`) 完全一致，只有当玩家碰触时才会消失（或变为半透明）。隐藏通道 (`H`) 是不可见的通行区域，玩家可以直接穿过。建议在密室入口附近放置一个微妙的视觉暗示（例如墙壁纹理略有不同，或者有一个金币悬浮在墙壁前方），引导细心的玩家去尝试。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **钥匙与锁门 (Key & Lock)** | Metroidvania 全系 | 玩家需要先找到钥匙才能打开特定的门。与隐藏通道结合，可以设计出"找到隐藏钥匙 - 打开主路上的锁门 - 进入新区域"的探索循环 |
| **可破坏方块 (Breakable Block)** | Mario, Mega Man | 玩家可以通过攻击或跳跃破坏特定方块，开辟新路径。与伪装墙的区别在于需要主动操作而非被动碰触 |

## 模板 7：踩敌跳板与双路分支 (Pogo Bounce & Branching Path)

**灵感来源**：Shovel Knight 的 Pogo/Shovel Drop 踩敌跳跃机制 + Mario 经典水管路径分支设计。

**设计理念**：关卡在中段分为上下两条路径。**上路**（安全路线）通过固定平台和少量地刺缓慢前进，没有金币奖励；**下路**（风险路线）是一条深渊上方的弹跳怪链——玩家必须连续踩踏 3 个弹跳怪作为跳板才能到达对岸，但沿途散落大量金币。两条路在终点前汇合。核心博弈是"安全但无聊 vs 危险但有趣"的路径选择心理，同时训练玩家"敌人不只是威胁，也是工具"的认知。渐进揭示：分支入口前有一个单独的弹跳怪+金币组合作为教学，让玩家先学会踩敌再做选择。

```
##############################
#............................#
#..........=...=...=.........#
#..........^.......^.........#
#M...o.............o.........#
##=..E..##.......o...o...o.G.#
#......##..=.=.=.............#
#.....##...^.^.^.....========#
#....##..............#########
#...##...o..o..o..o..........#
#..##....E..E..E.............#
#.##.........................#
##############################
```

**尺寸**：30 x 13

### 博弈解析

本模板覆盖了两个全新博弈维度。**路径分支**：玩家在第 5 行的岔路口面临选择——向上走安全平台（第 3 行的 `=` 序列，中间夹有地刺），或向下走弹跳怪链（第 11 行的 `E` 序列）。上路零奖励但几乎不会死；下路金币丰厚但需要连续精确踩踏。**敌人利用**：弹跳怪 (`E`) 在这里不是纯粹的威胁，而是必须踩踏的跳板。玩家需要理解"踩踏 = 弹跳"的物理规则，并在深渊上方连续执行 3 次。分支入口前的教学区（第 6 行左侧的单个 `E` + 金币 `o`）让玩家在安全环境下先体验一次踩敌弹跳。

### 动态参数建议

弹跳怪 (`E`) 建议设置为原地小幅跳跃（Jump 模式，高度 1 格，周期 1.2 秒），玩家踩踏时获得 3 格弹跳高度（高于普通跳跃的 3 格上限，提供额外的容错空间）。下路的 3 个弹跳怪间距 4 格（恰好等于水平极限跨越距离），要求玩家在弹跳最高点立刻向前移动。上路的地刺 (`^`) 放在平台之间的间隙中，间距 2 格，新手只需普通跳跃即可通过。金币 (`o`) 在下路形成抛物线引导线，暗示正确的踩踏节奏。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|
| **弹簧 (Spring)** | Sonic, Shovel Knight | 可控弹射方向的固定装置，与弹跳怪的区别在于弹簧是静态可预测的，弹跳怪有自己的运动节奏。两者组合可以创造"弹簧发射 → 踩敌转向 → 弹簧接力"的复杂弹射链 |
| **下砸攻击 (Pogo/Shovel Drop)** | Shovel Knight, Hollow Knight | 专用的向下攻击动作，踩敌时获得更高弹跳。当前系统中踩踏是被动触发的，增加主动下砸可以让玩家控制踩踏时机 |

## 模板 8：垂直坠井与视觉陷阱 (Vertical Descent & Visual Trap)

**灵感来源**：Downwell 的垂直下落闪避机制 + I Wanna Be The Guy 系列的视觉欺骗陷阱设计。

**设计理念**：这是一个纵向关卡——玩家从顶部出发，向下坠落穿过一个充满陷阱的竖井。竖井两侧交替伸出平台和地刺，玩家需要左右移动躲避。关键创新在于**视觉欺骗**：部分看起来像安全平台的位置实际上是伪装墙 (`F`)，踩上去会直接穿过掉入下方地刺；而部分看起来像墙壁的位置实际上是隐藏通道 (`H`)，穿过后可以找到金币密室。底部有弹跳平台 (`B`) 作为安全网，防止直接摔死。整体体验是"下落中的快速判断 + 被骗后的恍然大悟"。

```
W.....M.....W
W...........W
W..o........W
W..=........W
W.......F...W
W.......^...W
W...........W
W.=.........W
W.^.........W
W.........o.W
W.........=.W
W...F.......W
W...^.......W
W...........W
W........=..W
WH..o.o.o..HW
W.=======...W
W...........W
W.^.^.^.^.^.W
W....BBB....W
W.....G.....W
W###########W
```

**尺寸**：13 x 22

### 博弈解析

本模板覆盖了两个全新博弈维度。**垂直下落**：与传统的水平推进关卡完全不同，玩家的主要运动方向是向下。重力成为推进力，玩家只需控制左右移动和落点选择。两侧墙壁 (`W`) 形成竖井，平台 (`=`) 和地刺 (`^`) 从两侧交替伸出，创造出蛇形下落路径。**视觉欺骗**：第 5 行和第 12 行的伪装墙 (`F`) 在视觉上与普通平台 (`=`) 完全相同，但玩家踩上去会直接穿过。第一次被骗后，玩家会对所有平台产生怀疑，形成"信任危机"的心理博弈。第 16 行两侧的隐藏通道 (`H`) 则是反向欺骗——看起来是实心墙壁，实际上可以穿过进入金币密室，奖励勇于尝试的玩家。

### 动态参数建议

伪装墙 (`F`) 在视觉上必须与普通平台 (`=`) 完全一致（使用相同的颜色和纹理），玩家碰触时立刻变为半透明并失去碰撞体，0.5 秒后完全消失，3 秒后重生。隐藏通道 (`H`) 在视觉上与墙壁 (`W`) 一致，但没有碰撞体。弹跳平台 (`B`) 在底部提供 4 格弹跳高度，足以让玩家回到第 15 行的平台重新尝试。建议在伪装墙下方的地刺旁放置一个微妙的视觉线索（如地刺颜色略亮），让细心的玩家有机会识别陷阱。金币 (`o`) 在第 16 行的密室中作为探索奖励。

### 缺失机制分析

| 缺失机制 | 原型游戏 | 博弈乐趣 |
|----------|----------|----------|

## 拼装建议

这 8 个模板可以像搭积木一样组合使用。以下是几种推荐的拼装顺序：

**教学关卡（新手友好）**：模板 6（密室探索，教会玩家"到处试试"）然后接模板 1（地刺深渊，教会玩家精确跳跃）

**技巧关卡（中等难度）**：模板 2（Z 字攀爬）然后接模板 7（踩敌跳板与路径分支，教会玩家"敌人也是工具"）

**探索关卡（中等难度）**：模板 7（路径分支，奖励探索）然后接模板 6（密室探索，奖励好奇心）

**极限关卡（高难度）**：模板 5（崩塌逃亡）然后接模板 4（火焰走廊）

**心理战关卡（难度递进）**：模板 8（垂直坠井与视觉陷阱，打破信任）然后接模板 3（移动平台协奏，重建信任）

在 Unity 的 Custom Template Editor 中，你可以先粘贴第一个模板，然后在文本框中手动追加第二个模板（注意两个模板之间需要有一段连接地面），或者使用 Snippet Library 中的预设片段进行拼接。
