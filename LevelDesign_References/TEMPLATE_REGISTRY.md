# MarioTrickster — 关卡模板登记簿 (TEMPLATE_REGISTRY)

> **用途**：跨会话/跨账号去重的"外挂大脑"。每次让 AI 生成新模板前，AI 必须完整读取本文件。
>
> **去重粒度**：按"核心博弈组合"而非"类型"去重。同一类型（如地刺深渊）只要博弈手法不同就不算重复，鼓励同类型的深度探索。
>
> **搜集标准**：不止搜集极难毒图！有趣的、精妙的、创意独特的关卡设计同样欢迎。用难度标签区分。

## 语义指纹格式规范

新增模板时，必须按以下格式提取语义指纹：

```
T编号_[核心机制A] + [核心机制B]（= 玩家心理博弈体验）[难度:易/中/难/极难]
```

**示例**：
- `T01_静态地刺阵列 + 悬浮诱饵金币（= 被动风险选择）[难度:中]`
- `T04_上下火焰夹道 + 多摆锤错相位（= 纯时间窗口通过）[难度:难]`

**难度标签定义**：
| 标签 | 定义 |
|------|------|
| 易 | 新手友好，主要教学或探索，几乎不会死 |
| 中 | 需要一定技巧，可能死 1~3 次 |
| 难 | 需要精确操作和节奏把握，可能死 5~10 次 |
| 极难 | 毒图级别，需要反复练习和肌肉记忆 |

## 维护指南

**新增模板时**（5 步，60 秒）：

1. 在【完整登记表】追加一行，填写博弈组合摘要、难度标签、灵感来源
2. 在【快速复制区】的代码块中追加对应的一行摘要（含难度标签）
3. 在【已探索灵感来源】中追加本次参考的游戏+具体关卡
4. 如果该模板覆盖了【灵感池】中的某个维度，在该维度的状态列标注 `[已覆盖: T0x]`
5. 如果发现了新的缺失机制，追加到【缺失机制待办】中

**路线 A 的 AI 自动执行以上全部 5 步；路线 B 需要人工手动完成。**
