# MarioTrickster 美术管线白话速查表

> **前端极简自然语言模式已启用。** 你不用再背“菜单 1 / 菜单 2 / 菜单 3”，也不用判断自己该填哪个表。你只要直接用大白话告诉我你现在想达成什么结果，我来在后台判断该走哪段流程、该查哪个知识抽屉、该补哪份记忆、该下什么图纸。

这份速查表现在统一成更贴近真实工作的版本：**从零开始、喂书蒸馏、先在本地探索 30 张、去 Civitai 在线训练、练完后回本地验证、登记入库、再正式量产。** 其中最重要的澄清是：**常用主路径不是“本地训练 LoRA”，而是“在线训练 LoRA，本地验证效果”。**

| 你现在处在哪个阶段 | 你直接这样说就行 | 我会在后台自动做什么 |
|---|---|---|
| **还没开始，不知道先干嘛** | `我想从零开始做这套美术资产，你带我走。` | 自动判断你当前最缺的是教程蒸馏、概念探索、风格探索、训练准备还是直接量产，并只追问真正卡住执行的致命缺项。 |
| **要喂书 / 喂教程蒸馏规则** | `我上传了一本教程，帮我蒸馏进配方库并推送仓库。` | 阅读资料，提炼规则，写入 `prompts/PROMPT_RECIPES.md`，同步 `SESSION_TRACKER.md`，提交并推送。 |
| **要先探索风格，直接在本地出 30 张图去炼 LoRA** | `按这个参考风格，先在本地给我跑 30 张探索图，我拿去炼 LoRA。` | 识别为探索出图阶段，优先保证你能连续抽样、筛选、积累训练集，而不是把重点放在解释流程。 |
| **要问 Civitai 在线训练页怎么填** | `我要用 Civitai 练这个 LoRA，直接告诉我页面每一项怎么填。` | 按你的目标、素材量与训练类型，直接翻译成线上页面可照填的建议。 |
| **LoRA 练完了，要回本地验证** | `这个 LoRA 练完了，告诉我怎么在本地验证触发词、权重和污染。` | 直接给出本地验证顺序、测试重点、权重甜区试法与污染排查办法。 |
| **LoRA 已确认可用，要登记入库** | `我炼完 LoRA 了，文件名是 A，触发词是 B。帮我登记。` | 把 LoRA 写入 `prompts/PROMPT_RECIPES.md` 顶部名录，自动补齐默认继承范围、状态与备注，并同步追踪文件后提交推送。 |
| **已经要量产正式资产** | `做一组地刺的静态图。` | 自动调用默认 LoRA、实体蓝图、知识抽屉和 ControlNet 分流，直接输出 `LoRA / Prompt / ControlNet / KSampler` 四区图纸。 |

## 1. 如果你完全不知道从哪里开始

最简单的开场白就是下面这句。

> 我想从零开始做这套美术资产，你带我走。

这句话的意义，是把“判断当前阶段”的负担从你脑子里拿掉，交给我来做。如果你已经有参考图，我会把它识别成探索起点；如果你刚上传一本教程，我会把它识别成蒸馏任务；如果你已经有一批满意图，我会把它识别成准备上线训练；如果你已经在训练页面里，我会把它识别成参数辅导；如果你已经拿到 LoRA，我会把它识别成验证、登记或量产前夜。

## 2. 如果你现在是“喂书阶段”

你不需要再记“菜单 1”。以后只要像下面这样说就行。

> 我上传了一本教程，帮我蒸馏进配方库并推送仓库。

这句话等价于过去的“喂书蒸馏”。我会自动读取教程内容，把真正能落到出图层的规则拆进 **技法抽屉**，并完成文档落库、会话同步与仓库推送闭环。你不需要再背长版蒸馏指令，除非你想额外加限制条件。

## 3. 如果你现在是“先在本地出 30 张探索图，准备炼 LoRA”

你前端表达的应该是**生产目标**，而不是让我先回你一张说明单。以后更准确的白话开口方式如下。

> 按这个参考风格，先在本地给我跑 30 张探索图，我拿去炼 LoRA。

这句话等价于过去的“探索工单 / 抽卡工单 / 炼丹前样本积累阶段”。我会默认把这轮任务识别为**探索而非量产**，因此后台策略会优先帮你拿到一批可筛选、可积累、适合进训练集的样本，而不是一上来就按最终量产要求卡死结构。

| 你想做的事 | 你可以直接说的话 |
|---|---|
| **用参考图先找画风** | `我上传了参考图，先在本地帮我跑一批探索图，我想看看这个画风方向对不对。` |
| **想连续出 30 张做训练集** | `按这个参考风格，先在本地给我跑 30 张探索图，我拿去炼 LoRA。` |
| **只想先验证方向，不想定版** | `先别讲流程，按这个参考图在本地出一批探索图给我看方向。` |

## 4. 如果你现在要问“Civitai 在线训练页面怎么填”

这通常才是你的**主训练入口**。你不需要自己先把页面研究明白，最省算力的方式，就是一次把目标说完整，让我直接按页面逻辑帮你翻译。

| 你真正想要的帮助 | 你可以直接这样说 |
|---|---|
| **直接要一套 Civitai 可照填参数** | `我要用 Civitai 练这个 LoRA。数据集是 30 张图，目标是统一风格。直接告诉我各项怎么填。` |
| **让我按页面顺序带你填** | `我现在正打开 Civitai 训练页，你按页面顺序告诉我每一项该怎么选。` |
| **让我判断该按风格、角色还是对象思路去练** | `我要在 Civitai 练这个 LoRA。你帮我判断该按风格、角色还是对象去配。` |
| **让我检查已经填过的线上参数** | `这是我现在在 Civitai 填的参数，帮我直接检查哪里不合理。` |
| **让我根据失败结果反推设置问题** | `我这次 Civitai 练出来风格不稳 / 过拟合 / 泛化差，你根据我的设置直接告诉我怎么改。` |

如果你正在页面里手足无措，最短的说法可以直接是：

> 我现在在 Civitai 训练页，你一步一步告诉我该怎么填。

## 5. 如果你现在是“LoRA 练完后，要回本地验证”

不管你是在线练还是少数情况在本机自训，**练完后回本地验证**通常都是必须环节。你真正需要问的是：这个 LoRA 到底能不能用、甜区是多少、有没有污染、值不值得入库。

| 你真正想要的帮助 | 你可以直接这样说 |
|---|---|
| **让我告诉你本地验证顺序** | `这个 LoRA 练完了，告诉我怎么在本地一步一步验证。` |
| **让我专查触发词和权重甜区** | `这个 LoRA 练完了，帮我在本地验证触发词和权重甜区该怎么测。` |
| **让我判断有没有训练污染** | `这个 LoRA 出图老带脏特征 / 老像训练图。告诉我怎么本地验证是不是污染了。` |
| **让我判断能不能登记入库** | `这是这个 LoRA 的测试结果，你帮我判断现在能不能登记进项目库。` |
| **让我给出验证工单** | `这个 LoRA 刚练完，直接给我一套本地验证图纸，我照着测。` |

最短也可以直接说：

> 这个 LoRA 练完了，告诉我怎么在本地验证触发词、权重和污染。

## 6. 如果你真的要“本地自训 LoRA”

这个入口**仍然保留**，但它不是你的常用主路径，只有在你真的准备用自己显卡跑训练时才需要启用。

| 你真正想要的帮助 | 你可以直接这样说 |
|---|---|
| **让我判断该不该本地自训** | `我现在有 30 张图，显卡是 4070。你帮我判断这次该本地自训还是继续用 Civitai。` |
| **直接要一套本地自训参数** | `我要在自己电脑上训练这个 LoRA。数据集是 30 张图，显卡是 4070。直接给我一套能填的参数。` |
| **直接要一步一步的本地自训操作顺序** | `我要现在开始本地自训。按我的环境一步一步告诉我怎么做。` |
| **让我检查我现在填的本地自训参数** | `这是我现在的本地训练参数，帮我检查哪里有坑，直接告诉我该改哪几项。` |

如果你平时主要是在线练，那这一节可以当成**备用入口**看待。

## 7. 如果你现在是“LoRA 已炼完，准备入库”

你以后只要像平时说话一样告诉我就行，例如下面这句就是合法输入。

> 我炼完 LoRA 了，文件名是 `MarioTrickster_Style_epoch_10.safetensors`，触发词是 `trickster_style`。帮我登记进项目记忆库并推上仓库。

我收到后会默认把它当成项目风格资产来处理，而不是要求你再填一堆字段。只要文件名和触发词齐了，我就会自动把这张风格卡登记入库。若你还没试出最佳权重或专属负面词，我会先把这些字段写成 **`待验证`**，状态标成 **`验证中`**，不会因为少了非致命信息就卡住你。

## 8. 如果你现在是“直接量产资产”

你以后只要直接说你要做什么资产就行，例如下面这句就是合法输入。

> 做一组地刺的静态图。

这句话等价于过去的“正式派单量产”。我收到后会默认执行整套后台补全：先查当前可用的 LoRA 资产，再查实体蓝图，然后按任务需要静默调用 **解剖 / 透视 / 动画 / 光影 / 参数** 对应知识抽屉，最后按 **LoRA / Prompt / ControlNet / KSampler** 四区物理隔离下发图纸。你不需要先判断自己该走哪个菜单，也不需要先把镜头、材质、动作、ControlNet 类型一次性报全。

## 10. 如果你换号了 / 开新对话了，怎么一句话续上

你以后不用重新解释一大堆上下文，也不用自己总结项目历史。最省脑子的做法，就是**把仓库地址先给我，让我先读档；如果后面要继续推送，但当前环境缺认证，再补 token。**

| 你现在想做什么 | 你直接这样说 |
|---|---|
| **先把当前进度安全存起来** | `把当前进度存档到 GitHub，保证我下次换号开新对话也能接上。缺仓库地址或 token 你就直接问我。` |
| **换号后让 AI 先读档接上** | `我换号了，这是我的 GitHub 仓库地址：XXX。你先读档接上当前进度继续；如果后面要 push 还缺 token，你再只问我要那个。` |
| **这次大概率还要重新认证** | `我换号了，这是仓库地址：XXX。这次 GitHub 认证也得重配，你先读档，需要 token 时一步一步告诉我。` |

最短版你就记住下面两句。

> 存档时说：`先把当前进度写进仓库并推送，保证我下次换号开新对话也能接上。缺仓库地址或 token 就直接问我。`
>
> 读档时说：`我换号开新对话了。这是我的 GitHub 仓库地址：XXX。你先读档接上当前进度继续做；如果要写回仓库但缺认证，你再告诉我现在要补 token。`

所以规则只有一条：**读档先给仓库地址，继续写回 GitHub 时按需补 token。**

## 11. 为了避免反复追问，最省算力的说法

如果你希望我尽量一次答准，问训练问题时最好顺手把下面几类信息一起说出来。但你不用机械填表，只要自然地说进一句话里就行。

| 你最好顺手带上的信息 | 例子 |
|---|---|
| **平台** | `Civitai`、`我自己的电脑`、`我已经在本地验证了` |
| **目标类型** | `统一风格`、`角色`、`道具`、`场景`、`动作` |
| **素材量** | `30 张图`、`80 张图` |
| **环境** | `我已经在 Civitai 页面里了`、`显卡是 4070` |
| **你当前卡住的点** | `不知道 epoch 怎么定`、`不知道 repeats 填多少`、`不知道是不是污染了` |

例如一句就够：

> 我要用 Civitai 练统一风格 LoRA，手上有 30 张图，我已经打开训练页了，但不知道 repeats 和 epoch 怎么定。你直接按页面顺序告诉我怎么填。

## 12. 后台默认铁律

| 后台默认铁律 | 实际执行方式 |
|---|---|
| **自然语言优先** | 禁止把用户重新拖回表单、填空题或多分支口令。 |
| **100% 上下文补全** | 先读 `prompts/PROMPT_RECIPES.md`，自动继承 LoRA 名录、风格合同、实体蓝图与技法抽屉。 |
| **真实主路径优先** | 优先把训练相关问题识别成“探索出图 → 在线训练 → 本地验证 → 登记入库”，而不是默认误判成“本地自训”。 |
| **双轨架构不变** | 上半部技法库负责 How，下半部实体库负责 What；出图必须是“实体蓝图 × 技法抽屉”的十字交叉。 |
| **四区物理隔离** | 风格调用、最终提示词、结构约束、采样参数必须分区输出，绝不揉成一坨长 Prompt。 |
| **冲突熔断优先级** | 永远遵守 `物理防滑步 > 空间透视正确 > 光影材质表现`。 |
| **非必要绝不追问** | 只有在缺少会让任务根本无法执行的致命条件时，才允许用一句话反问。 |

所以你以后不必再记住“我现在是不是菜单 1、菜单 2 还是菜单 3”。你只需要记住一句更人类的话：**直接说你此刻想达成什么结果。** 我会负责把它翻译成后台流程。
