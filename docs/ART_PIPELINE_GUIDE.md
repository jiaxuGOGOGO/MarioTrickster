# MarioTrickster 美术管线操作菜单 (大白话版)

> **💡 你现在该做什么？**
> 这个文档不是单行道。以下 **3 件事情你可以随时做、同时做**。
> 你可以先定画风再去出图，也可以先发一本好教程给我蒸馏，甚至出图出到一半想换画风推翻重来都可以。
> 选你现在想做的一件事，直接复制对应的指令发给我。

---

## 菜单 1：确立或更换画风 (随时可做)

在批量做游戏资产前，我们要先定一张"样板间照片"（概念锚点）。这张图会决定整个游戏的色调和画风。以后做任何东西，都以这张图为准。
如果你做了一半看腻了，也可以随时用这个流程换一个新画风，物理手感不会受任何影响。

### 💬 怎么找我：
> "Manus，我要[确立 / 更换]本项目的基准画风。我上传了一张参考图（或者我想要的风格是 XXX）。请根据我的偏好，给我派发一张『概念锚点 Mockup』工单。我出图满意后，会将图存入 `Assets/Art/Reference/Reference_Anchor.png`。"

### 🛠️ 你接下来要干嘛：
1. 拿着我给你的 Prompt，去本地的 ComfyUI 出图。
2. 一直抽卡，直到抽出一张你觉得"这就是我想要的游戏画面"的横版截图。
3. 记下这张图的 **Seed 值**。
4. 把这张图保存为 `Assets/Art/Reference/Reference_Anchor.png`。

---

## 菜单 2：喂书蒸馏 (有好教程时随时可做)

如果你在网上看到一本很牛的美术教程、设定集，或者一套很好的 ComfyUI 出图参数，你可以发给我。
我会把里面好的技巧"抄"下来，补充到我们的配方库里。但我会把关，绝不吸收会破坏物理手感（比如导致滑步）的错误规则。

### 💬 怎么找我：
> "Manus，我上传了一本新教程/设定集。请执行『防污染 PR 合并』流程：提取其中的优秀 Prompt 和参数，对比 `ART_BIBLE`，以『最防滑步』为唯一准则解决冲突，绝不缝合。然后发 PR 给我 Review。"

### 🛠️ 你接下来要干嘛：
1. 喝杯咖啡，等我读完 PDF。
2. 我会提一个 PR（Pull Request）给你。
3. 你去 GitHub 看看我加了什么新规矩，觉得没问题就点 Approve 合并。

## 菜单 3：批量生产具体资产 (日常循环)

当你的画风（菜单 1）定好之后，你就可以开始批量做游戏里真正要用的东西了（比如"主角跑步"、"弹跳怪"）。

### 💬 怎么找我：
> "Manus，我要做【主角跑步动画】。请帮我算好帧数，输出中英文 Prompt 和参数，存到配方库里，然后把 Prompt 发给我。"

### 🛠️ 你接下来要干嘛 (端到端闭环)：
1. **本地出图**：拿着我给你的 Prompt 去 ComfyUI 出图。**必须把之前存的 `Reference_Anchor.png` 拖进 IPAdapter 里垫图**。
2. **反馈调整**：如果不满意，把有瑕疵的图发给我，告诉我哪里不对（比如"腿画歪了"），我帮你改 Prompt。
3. **导入 Unity**：把满意的图（纯色背景去底后的 PNG）拖进 Unity 的 `Assets/Art/` 对应文件夹里。
4. **一键切图**：在 Unity 里点顶部菜单 `MarioTrickster/Art Pipeline/一键工业化切图`。它会自动帮你切好帧，并把重心锁死防滑步。
5. **测试验收**：拖进场景里跑一跑，点 `一键合规巡检`。全绿通过后，告诉我："【主角跑步动画】测试通过，请 Push 代码。"

## 💻 附录：你需要准备的本地软件

为了执行上述菜单，你的电脑上需要：
1. **ComfyUI**（推荐）：用于出图。
2. **必需模型**：
   - 大模型：`sd_xl_base_1.0.safetensors` 或 `Animagine XL`（放进 `models/checkpoints`）
   - LoRA：`pixel-art-xl-v1.1.safetensors`（放进 `models/loras`）
   - 插件：必须安装 `IPAdapter` 节点（用于吸取画风）。
