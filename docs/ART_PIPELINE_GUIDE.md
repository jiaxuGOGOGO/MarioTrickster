# MarioTrickster 美术管线白话速查表

> **前端极简自然语言模式已启用。** 你不用再背“菜单 1 / 菜单 2 / 菜单 3”的口令，也不用再判断自己该填哪个表。你只要直接用大白话告诉我你现在想干什么，我来在后台判断该走哪一段流程、该查哪个知识抽屉、该补哪份记忆、该下什么工单。

这份速查表现在补成了**从零开始的完整白话入口**。也就是说，无论你现在是在“还没开始、想先喂书、想先抽 30 张探索图、想问本地训练参数、想问 Civitai 页面该怎么填、准备炼 LoRA、已经炼完要登记、还是已经要量产资产”的哪个阶段，你都能直接开口，我来自动接住。

| 你现在处在哪个阶段 | 你直接这样说就行 | 我会在后台自动做什么 |
|---|---|---|
| **还没开始，不知道先干嘛** | `我想从零开始做这套美术资产，你带我走。` | 自动判断你当前最缺的是教程蒸馏、概念探索、风格探索、训练准备还是直接量产，并只追问真正卡住执行的致命缺项。 |
| **要喂书 / 喂教程蒸馏规则** | `我上传了一本教程，帮我蒸馏进配方库并推送仓库。` | 阅读资料，提炼规则，写入 `prompts/PROMPT_RECIPES.md`，同步 `SESSION_TRACKER.md`，提交并推送。 |
| **要先探索风格，出 30 张图去炼 LoRA** | `基于这个参考风格，给我一套本地跑图工单，我要连续抽 30 张拿去炼 LoRA。` | 输出探索期工单，静默组合风格与参数策略，优先保证你能高效抽样、筛选、积累训练集。 |
| **要问本地训练参数或操作** | `我要在本地训练这个 LoRA，直接给我一套能填的参数和操作顺序。` | 根据你的显卡、数据量和目标，直接给出可执行的本地训练建议与操作顺序。 |
| **要问 Civitai 训练参数或操作** | `我要用 Civitai 训练这个 LoRA，直接告诉我各项怎么填。` | 按你的目标与素材情况，直接给出适合线上训练页面的填写建议。 |
| **LoRA 已炼完，要登记入库** | `我炼完 LoRA 了，文件名是 A，触发词是 B。帮我登记。` | 把 LoRA 写入 `prompts/PROMPT_RECIPES.md` 顶部名录，自动补齐默认继承范围、状态与备注，并同步追踪文件后提交推送。 |
| **已经要量产正式资产** | `做一组地刺的静态图。` | 自动调用默认 LoRA、实体蓝图、知识抽屉和 ControlNet 分流，直接输出 `LoRA / Prompt / ControlNet / KSampler` 四区工单。 |

## 1. 如果你完全不知道从哪里开始

最简单的开场白就是下面这句。

> 我想从零开始做这套美术资产，你带我走。

这句话的意义，是把“判断当前阶段”的负担从你脑子里拿掉，交给我来做。如果你已经有参考图，我会把它识别成探索起点；如果你刚上传一本教程，我会把它识别成蒸馏任务；如果你已经有一批满意图，我会把它识别成炼 LoRA 前夜；如果你已经准备点开训练页面，我会识别成参数与操作辅导；如果你已经给出明确资产目标，我会判断是否可以直接量产。

## 2. 如果你现在是“喂书阶段”

你不需要再记“菜单 1”这个名字。以后只要像下面这样说就行。

> 我上传了一本教程，帮我蒸馏进配方库并推送仓库。

这句话等价于过去的“喂书蒸馏”。我会自动读取教程内容，把真正能落到出图层的规则拆进 **技法抽屉**，并完成文档落库、会话同步与仓库推送闭环。你不需要再背长版蒸馏指令，除非你想额外加限制条件。

## 3. 如果你现在是“先出 30 张图探索风格，准备炼 LoRA”

你不需要再说“我要走菜单 2”。以后你只要直接说结果目标，例如下面这句。

> 基于这个参考风格，给我一套本地跑图工单，我要连续抽 30 张拿去炼 LoRA。

这句话等价于过去的“探索工单 / 抽卡工单 / 炼丹前样本积累阶段”。我会默认把这轮任务识别为**探索而非量产**，因此后台策略也会不同：我会优先帮你拿到一批风格稳定、可筛选、适合进训练集的样本，而不是一上来就按最终量产要求卡死结构。

| 你想做的事 | 你可以直接说的话 |
|---|---|
| **用参考图先找画风** | `我上传了参考图，帮我出一套探索工单，我要先抽风格。` |
| **想连续抽 30 张做训练集** | `基于这个参考风格，给我一套能连续抽 30 张的本地工单，我要炼 LoRA。` |
| **只想先验证方向，不想定版** | `先帮我跑探索方案，我想看看这个画风方向对不对。` |

## 4. 如果你现在要问“本地训练参数或操作怎么配”

如果你不想反复追问，最稳的说法是一次把**平台、目标、素材量、硬件**一起说出来。你不需要自己先研究一堆参数名，只要把目标说清，我来翻译。

| 你真正想要的帮助 | 你可以直接这样说 |
|---|---|
| **让我帮你判断该不该本地练** | `我现在有 30 张图，显卡是 4070，目标是做统一风格 LoRA。你帮我判断该本地练还是上 Civitai，并直接给建议。` |
| **直接要一套本地训练参数** | `我要在本地训练这个 LoRA。数据集是 30 张图，目标是统一风格，显卡是 4070。直接给我一套能填的参数。` |
| **直接要一步一步的本地操作顺序** | `我要现在开始本地训练。按我的环境一步一步告诉我该怎么做，不要讲原理，直接讲操作。` |
| **让我检查我现在填的本地参数** | `这是我现在的本地训练参数，帮我检查哪里有坑，直接告诉我该改哪几项。` |
| **让我排查欠拟合 / 过拟合** | `这版 LoRA 出图太淡 / 太糊 / 太像训练图。结合我的参数帮我判断原因，并直接告诉我怎么改。` |
| **让我定 trigger word 和命名** | `这个 LoRA 我该怎么起触发词最稳，顺便给我建议文件命名。` |

如果你当时手边只有一张参数页截图，也完全可以直接说：

> 这是我本地训练页面的参数截图，帮我直接检查并告诉我该改哪几项。

## 5. 如果你现在要问“Civitai 训练页面各项怎么填”

你也不需要自己先把 Civitai 页面研究明白。最省算力的方式，就是一次把目标说完整，让我直接按页面逻辑帮你翻译。

| 你真正想要的帮助 | 你可以直接这样说 |
|---|---|
| **直接要一套 Civitai 可照填参数** | `我要用 Civitai 训练这个 LoRA。数据集是 30 张图，目标是统一风格。直接告诉我各项怎么填。` |
| **让我按页面顺序带你填** | `我现在正打开 Civitai 训练页，你按页面顺序告诉我每一项该怎么选。` |
| **让我判断 Civitai 上该选哪种训练类型** | `我要在 Civitai 练这个 LoRA。你帮我判断该选风格、角色还是对象类思路，并解释成我能直接照做的选法。` |
| **让我检查你已经填过的线上参数** | `这是我现在在 Civitai 填的参数，帮我直接检查哪里不合理。` |
| **让我根据失败结果反推设置问题** | `我这次 Civitai 练出来风格不稳 / 过拟合 / 泛化差，你根据我的设置帮我反推该怎么改。` |

如果你正在页面里手足无措，最短的说法可以直接是：

> 我现在在 Civitai 训练页，你一步一步告诉我该怎么填。

## 6. 如果你现在是“LoRA 已炼完，准备入库”

你以后只要像平时说话一样告诉我就行，例如下面这句就是合法输入。

> 我炼完 LoRA 了，文件名是 `MarioTrickster_Style_epoch_10.safetensors`，触发词是 `trickster_style`。帮我登记进项目记忆库并推上仓库。

我收到后会默认把它当成项目风格资产来处理，而不是要求你再填一堆字段。只要文件名和触发词齐了，我就会自动把这张风格卡登记入库。若你还没试出最佳权重或专属负面词，我会先把这些字段写成 **`待验证`**，状态标成 **`验证中`**，不会因为少了非致命信息就卡住你。

## 7. 如果你现在是“直接量产资产”

你以后只要直接说你要做什么资产就行，例如下面这句就是合法输入。

> 做一组地刺的静态图。

这句话等价于过去的“正式派单量产”。我收到后会默认执行整套后台补全：先查当前可用的 LoRA 资产，再查实体蓝图，然后按任务需要静默调用 **解剖 / 透视 / 动画 / 光影 / 参数** 对应知识抽屉，最后按 **LoRA / Prompt / ControlNet / KSampler** 四区物理隔离下发工单。你不需要先判断自己该走哪个菜单，也不需要先把镜头、材质、动作、ControlNet 类型一次性报全。

## 9. 为了避免反复追问，最省算力的说法

如果你希望我尽量一次答准，问训练问题时最好顺手把下面几类信息一起给出来。但你不用机械填表，只要自然地说进一句话里就行。

| 你最好顺手带上的信息 | 例子 |
|---|---|
| **平台** | 本地 / Civitai |
| **目标类型** | 统一风格 / 角色 / 道具 / 场景 / 表情 / 动作 |
| **素材量** | 例如 `30 张图`、`80 张图` |
| **硬件或环境** | 例如 `4070`、`我已经在 Civitai 页面里了` |
| **你当前卡住的点** | 例如 `不知道 epoch 怎么定`、`不知道 repeats 填多少`、`不知道该选什么训练类型` |

例如一句就够：

> 我要在本地 4070 上训练统一风格 LoRA，手上有 30 张图，不知道 epoch 和 repeats 怎么定。你直接给我一套建议。

## 10. 后台默认铁律

| 后台默认铁律 | 实际执行方式 |
|---|---|
| **自然语言优先** | 禁止把用户重新拖回表单、填空题或多分支口令。 |
| **100% 上下文补全** | 先读 `prompts/PROMPT_RECIPES.md`，自动继承 LoRA 名录、风格合同、实体蓝图与技法抽屉。 |
| **双轨架构不变** | 上半部技法库负责 How，下半部实体库负责 What；出图必须是“实体蓝图 × 技法抽屉”的十字交叉。 |
| **训练问题也走白话入口** | 只要你说清平台、目标和大致素材情况，我就负责把自然语言翻译成可执行的训练设置建议。 |
| **四区物理隔离** | 风格调用、最终提示词、结构约束、采样参数必须分区输出，绝不揉成一坨长 Prompt。 |
| **冲突熔断优先级** | 永远遵守 `物理防滑步 > 空间透视正确 > 光影材质表现`。 |
| **非必要绝不追问** | 只有在缺少会让任务根本无法执行的致命条件时，才允许用一句话反问。 |

所以你以后不必再记住“我现在是不是菜单 1、菜单 2 还是菜单 3”。你只需要记住一句更人类的话：**直接说你此刻想达成什么结果。** 我会负责把它翻译成后台流程。
