# MarioTrickster 角色素材调教指南（大白话版）

这份指南专为你（项目主导者）编写。当你在网上买了一套精美的像素角色素材，想把它完美地穿到游戏里的 Mario 身上时，可能会遇到各种“看起来很怪”的问题。不用慌，这篇指南会用最通俗的语言，告诉你怎么把它们调教得服服帖帖。

---

## 核心原则：物理是骨架，动画是衣服

在调教之前，你只需要记住一句话：**“物理是骨架，动画是衣服。”**

游戏里的角色移动、跳跃高度、重力下落，这些都是“物理骨架”决定的（也就是碰撞体白框）。而你导入的图片，只是穿在骨架外面的“衣服”。
- 衣服太大？把衣服缩水（调 Scale）。
- 衣服穿错了？换一件（调帧）。
- **绝对不要为了迁就衣服，去打断骨架（不要改碰撞体尺寸、不要改重力、不要改代码）。**

---

## 常见问题 1：角色导入后变得巨大无比

**现象**：应用素材后，角色图片比碰撞体白框大出好几倍，像个巨人。
**原因**：项目的标准是 32 像素 = 1 格（PPU=32）。如果你买的素材是 64×64 甚至 128×128 像素，系统按原比例渲染，就会显得很大。

**调教方法（绿线操作，随便改）**：
1. 在 Unity 顶部点击 **Scene** 标签页（不要在 Game 视图调）。
2. 在左侧 Hierarchy 面板，展开 `Mario`，选中它下面的 **`Visual`** 子节点。
3. 在右侧 Inspector 面板，找到 **Transform** 组件。
4. 把 **Scale** 的 X 和 Y 值改小（比如从 1 改成 0.5）。
5. 看着 Scene 视图里的碰撞体白框，把 Scale 调到图片刚好略大于白框即可。

> **避坑警告**：千万不要去改 `Mario` 根物体的 Scale，也不要去改 `PhysicsMetrics` 里的碰撞体尺寸。只改 `Visual` 的 Scale！

---

## 常见问题 2：新加一个动作，之前的动作全没了

**现象**：本来已经配好了 Idle 和 Walk，后来又单独拖了一个 Run 素材应用给角色，结果角色只会跑了，站着也在跑。
**原因**：系统的“一键换装”是整体覆盖的。你单独应用一张图，系统会以为你要把角色变成一个“只会循环播放这张图”的普通物体。

**调教方法**：
1. **必须打包应用**：把你想用的所有状态图片（比如 `Idle.png`、`Run.png`、`Jump.png`、`Fall.png`）放在**同一个文件夹**里。
2. 选中 Mario，打开 Apply Art 窗口。
3. 把**整个文件夹**拖进去，或者选中文件夹里的所有图片，一起应用。
4. 系统会自动把它们分门别类，挂上状态动画系统。

---

## 常见问题 3：放了 Walk 和 Run，结果跑起来没动画

**现象**：文件夹里同时放了 `Walk.png` 和 `Run.png`，应用后角色跑动时播放的却是站立动画。
**原因**：在系统眼里，Walk 和 Run 是同一个意思（都是跑步状态）。两个文件抢同一个位置，系统懵了，最后干脆塞了站立帧进去兜底。

**调教方法**：
1. **二选一**：删掉其中一个。如果你觉得 Walk 的慢走动作更好看，就留下 `Walk.png`，删掉 `Run.png`。
2. 系统会自动把 Walk 识别为跑步动作，移动时就会正常播放了。

---

## 常见问题 4：跳跃/下落时，角色在空中鬼畜循环

**现象**：角色跳起来时，在空中反复做“蹲下→起跳→伸展”的动作；或者下落时在空中疯狂后空翻。
**原因**：你买的商业素材，Jump 和 Fall 往往是一整套完整的动作（比如 10 帧）。但游戏里，角色在空中上升的时间只有零点几秒。系统默认是循环播放的，所以 10 帧播不完就从头再来，看起来就像在鬼畜。

**调教方法（精修帧）**：
我们需要把一整套动作，精简成“定格姿势”。
1. 选中 `Mario` -> `Visual`，在 Inspector 找到 **Sprite State Animator** 组件。
2. 展开 **Jump** 状态组：
   - 把 **Loop**（循环）的勾**去掉**。
   - 把 **Frames** 的 Size（数量）改成 2。
   - 点开 Element 0 和 Element 1，从右下角的小圆点里，选出角色**身体向上伸展**的那两帧（比如 `Jumping_F3` 和 `F4`）。不要选蹲下准备的帧。
3. 展开 **Fall** 状态组：
   - 把 **Loop** 的勾**去掉**。
   - 把 **Frames** 的 Size 改成 2。
   - 选出角色**手脚张开往下掉**的那两帧（比如 `Falling_F0` 和 `F1`）。不要选落地翻滚的帧。
4. **Idle** 和 **Run** 保持 Loop 开启，它们本来就需要循环。

这样调完，角色跳跃时就会保持一个干净利落的腾空姿势，手感瞬间变好。

---

## 常见问题 5：素材包里有爬墙、游泳、二段跳，怎么用？

**现象**：买的素材包太丰富了，有几十个动作，不知道怎么适配。
**原因**：当前项目只支持 4 个核心物理状态：站立（Idle）、跑（Run）、上升（Jump）、下落（Fall）。

**调教方法**：
1. **只取所需**：从素材包里挑出这 4 个状态对应的图片，放进文件夹应用。
2. **平替使用**：如果你觉得普通的 Jump 不好看，可以把 `Gliding_Jump.png` 改名叫 `Jump.png` 用，换个口味。
3. **暂存备用**：爬墙、游泳、翻滚这些动作，目前游戏里没有对应的物理机制，穿上也没用。把它们留在硬盘里，等以后项目加了新玩法再拿出来用。

---

## 总结：你的日常调教三步曲

以后每次换新皮肤，只需熟练默念这三步：
1. **凑齐四件套**：把 Idle、Run、Jump、Fall 四张图放进一个文件夹，整体应用。
2. **缩放合身**：调 Visual 的 Scale，让图片刚好包住碰撞体白框。
3. **掐头去尾**：把 Jump 和 Fall 的 Loop 关掉，只留腾空和下坠的 1~2 帧。

搞定这三步，任何商业素材都能在你的项目里跑得丝般顺滑！
