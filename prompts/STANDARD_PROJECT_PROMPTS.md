# MarioTrickster 标准提示词模板库

> **用途**：以后所有“更新 / 升级 / 优化 / 续接 / 规划 / 实现 / 蒸馏 / 验证 / 复盘”类新对话，优先从本文件复制模板，而不是临时自由发挥。  
> **配套协议**：使用前默认受 `docs/STANDARD_CONVERSATION_PROTOCOL.md` 约束。  
> **使用原则**：先选最接近当前任务的模板，再把你的个性化问题放到模板末尾的【我的补充要求】里；不要删掉模板中的协议段落。

---

## 1. 通用总模板（以后默认优先用这个）

> 适用于：你不确定该用哪一类模板，但能确认这轮属于“项目更新 / 优化 / 升级 / 续接”。

```text
我要继续维护 / 升级 / 优化 MarioTrickster 项目。

仓库地址：[必填，若本轮涉及读档或落库建议提供]
token：[如本轮可能提交/推送，建议开场直接提供]
本次任务：[用一句话写清本轮目标]

请严格按 MarioTrickster 标准对话协议执行，要求如下：
1. 先读取 `SESSION_TRACKER.md`，再按任务类型补读相关专项文档，不要让我手动复述长上下文。
2. 先判断当前阶段、主阻塞、当前默认入口、这次任务边界，再开始计划。
3. 先给出“当前判断 + 分阶段计划 + 预计修改文件/文档范围 + 验证方式”，不要直接开干。
4. 如果需求有歧义、范围不清或证据不足，先澄清，禁止脑补。
5. 执行时遵守“先窄入口，后扩范围”的原则，不要无故发散成大而全重构。
6. 完成后必须明确汇报：做了什么、改了哪些文件、已验证什么、未验证什么、当前风险是什么。
7. 凡是形成可复用结论的内容，都必须落库到对应真相源；未落库不算真正完成。
8. 如果本轮能提交并推送，就在完成后处理；如果当前条件不足，请明确列出待落库事项和缺口。

【我的补充要求】
- [在这里填写你本轮的特殊偏好、关注点或额外问题]
```

## 10. 使用建议

以后默认建议如下：

| 场景 | 建议使用的模板 |
|---|---|
| **不确定用哪种，但确定是项目更新** | **1. 通用总模板** |
| **换号 / 换窗口续接** | **2. 新对话极速续接模板** |
| **协议、文档、治理类深改** | **3. 项目级升级 / 深度改造模板** |
| **白盒关卡推进** | **4. 关卡主线 / 白盒推进模板** |
| **LoRA、本地验证、量产派单** | **5. 美术验证 / LoRA / 量产模板** |
| **蒸馏教程并入库** | **6. 教程蒸馏 / 规则入库模板** |
| **修 Bug / 做回归治理** | **7. Bug 修复 / 回归验证模板** |
| **结束前存档收口** | **8. 会话收口 / 存档模板** |

如果你懒得区分，**长期默认就用 1 号模板**。它已经把项目衔接、质量、效率与防失控需要的骨架都保住了。
