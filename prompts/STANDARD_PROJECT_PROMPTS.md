# MarioTrickster 标准提示词模板库

> **用途**：以后所有“更新 / 升级 / 优化 / 续接 / 规划 / 实现 / 蒸馏 / 验证 / 复盘”类新对话，优先从本文件复制模板，而不是临时自由发挥。  
> **配套协议**：使用前默认受 `docs/STANDARD_CONVERSATION_PROTOCOL.md` 约束。  
> **使用原则**：先选最接近当前任务的模板，再把你的个性化问题放到模板末尾的【我的补充要求】里；不要删掉模板中的协议段落。

---

## 0. 项目固定信息（以后默认不再重复手填）

下列信息属于 **MarioTrickster 的稳定上下文**，后续标准提示词默认直接沿用，除非你明确说明仓库迁移、分支切换或路径变化。

| 固定项 | 固定值 | 用途 |
|---|---|---|
| **项目名** | `MarioTrickster` | 锁定当前唯一项目上下文 |
| **远程仓库** | `https://github.com/jiaxuGOGOGO/MarioTrickster.git` | 锁定默认 GitHub 仓库 |
| **本地路径** | `/home/ubuntu/MarioTrickster` | 锁定默认工作目录 |
| **默认分支** | `master` | 锁定默认提交与校验分支 |
| **默认首读文档** | `SESSION_TRACKER.md` | 锁定新对话首读真相源 |
| **协议文档** | `docs/STANDARD_CONVERSATION_PROTOCOL.md` | 锁定项目级协作骨架 |
| **工作流说明** | `AI_WORKFLOW.md` | 锁定提问、落库、提交与远程校验规则 |

这意味着，后续标准新对话原则上**不需要你再手动重复仓库地址、本地路径、默认分支和基础入口**。对于一般的继续开发 / 优化 / 修复场景，你通常只需要补充 **本次任务** 与 **本次反馈 / 新变化**。只有当本轮涉及新的凭据、跨仓库协作、切分支或切目录时，才在模板的补充区显式声明。**例外情况** 是“换号 / 换窗口后的新对话读档续接”：这类模板为了避免读档后在提交、推送或远程校验阶段被中途打断，默认建议把 **可用 token** 也一并给出。

## 10. 使用建议

以后默认建议如下。

| 场景 | 建议使用的模板 |
|---|---|
| **不确定用哪种，但确定是项目更新** | **1. 通用总模板** |
| **换号 / 换窗口续接** | **2. 新对话极速续接模板** |
| **协议、文档、治理类深改** | **3. 项目级升级 / 深度改造模板** |
| **白盒关卡推进** | **4. 关卡主线 / 白盒推进模板** |
| **LoRA、本地验证、量产派单** | **5. 美术验证 / LoRA / 量产模板** |
| **蒸馏教程并入库** | **6. 教程蒸馏 / 规则入库模板** |
| **修 Bug / 做回归治理** | **7. Bug 修复 / 回归验证模板** |
| **结束前存档收口** | **8. 会话收口 / 存档模板** |

如果你懒得区分，**长期默认就用 1 号模板**。它已经把项目身份、工作目录、默认分支、读档入口、落库纪律与远程校验骨架都固化进去了。对于一般续接，你原则上只需要补 **“本次任务”** 与 **“本次反馈 / 新变化”**；如果是 **换号 / 换窗口后的新对话读档续接**，则优先改用 **2 号模板**，并把 **可用 token** 一并带上，避免后续推送阶段中断。
