# MarioTrickster 标准提示词与协作协议深度改造交付报告

## 1. 交付结论

本次已完成一轮**项目级深改**，目标是把 MarioTrickster 后续所有“更新 / 升级 / 优化项目”的新对话，从依赖临场发挥的随意开局，改造成依赖**固定协议 + 标准模板 + 仓库内回写规则**的稳定协作方式。

本次深改的核心判断是：项目当前真正缺的不是更多泛化研究，也不是再造一套新的平行体系，而是把已有的高价值规则、当前阶段口径、窄入口原则、验证闸门与落库纪律，**统一折叠成一个可以长期重复使用的新对话骨架**。这轮交付已经把该骨架正式落库，并接到了项目首页与当前会话真相源上。

## 3. 新体系解决了什么问题

| 问题 | 本次改造后的解决方式 |
|---|---|
| **新对话过于随意，容易漏条件** | 统一要求先套标准模板，再填本轮任务 |
| **换窗口后上下文恢复不稳定** | 固定为先读 `SESSION_TRACKER.md`，再按任务类型补读专项文档 |
| **任务容易从窄入口发散成大重构** | 协议中明确“先窄入口，后扩范围”，并要求先给阶段判断与范围 |
| **AI 容易边做边猜** | 把“模糊即停，禁止脑补”写成硬规则 |
| **可复用结论停留在聊天里** | 把“凡可复用结论必须落库”写成完成条件 |
| **验证不足却被说成完成** | 引入现状、范围、验证、落库四道闸门 |
| **用户个性化需求会冲掉固定纪律** | 单独设置【我的补充要求】插槽，让自定义问题与标准骨架并存 |

## 4. 你以后实际怎么用

以后默认请这样操作：

1. 如果你要继续任何项目更新、升级、优化任务，**优先打开** `prompts/STANDARD_PROJECT_PROMPTS.md`。  
2. 如果你懒得区分场景，**长期默认直接复制其中的“1. 通用总模板”**。  
3. 如果是换账号、换窗口续接，就用“2. 新对话极速续接模板”。  
4. 如果是协议、文档、治理类大改，就用“3. 项目级升级 / 深度改造模板”。  
5. 你的个人问题、偏好、反馈与附加限制，统一写到模板末尾的 **【我的补充要求】** 里。

这意味着，以后不再建议用“随便帮我看看”“你先自己发挥优化一下”这类宽泛开场白直接启动项目级任务，因为这类开场白最容易把范围、阶段和验证边界冲散。

## 5. 当前阶段下的推荐默认入口

本次协议深改**没有改变 MarioTrickster 当前阶段判断**，只是把它写得更稳定了。当前默认口径仍是：

| 主线 | 默认入口 | 说明 |
|---|---|---|
| **关卡主线** | `先继续白盒关卡，不等最终美术。` | 继续保证项目主线推进 |
| **美术支线** | `先做 trickster_style 本地验证，再决定切片量产。` | 当前真正主阻塞仍是 LoRA 本地验证缺失 |
| **桥接层** | 在已有命名资产需求时推进接回定义与首批量产 | 防止又回到无上游约束的泛探索 |

也就是说，**你要的是更稳的开局方式，不是推翻当前阶段结论重来一遍**；本次深改已经保持了这一点。
