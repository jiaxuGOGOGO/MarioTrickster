# 菜单 2.5 → 菜单 3 提示词衔接优化说明

本次我已经把仓库中的 `docs/ART_PIPELINE_GUIDE.md` 更新到 **V4.3**，核心目的是把你现在最容易混淆的两件事彻底拆开：**训练阶段的提示词** 和 **量产阶段的提示词** 不是同一个层级，不能拿来互相替代。

| 层级 | 你看到的位置 | 谁负责 | 本质作用 | 是否直接喂给模型 |
|---|---|---|---|---|
| **训练测试词** | 菜单 2.5 的 `Sample Media Prompts` | 你 | 让训练平台在每个 Epoch 抽样，检查 LoRA 有没有学到画风 | 否 |
| **换轨口令** | 你发给 Manus 的“LoRA 已就绪”那句话 | 你 | 告诉我：从菜单 2 退出，正式切到菜单 3 | 否 |
| **资产需求口令** | 菜单 3 的“我要做什么资产”模板 | 你 | 只说明要做什么、静态还是动画、是否需要动作或透视约束 | 否 |
| **最终生图 Prompt** | 我下发工单里的 `Text Prompt` 节点 | 我 | 把 `LoRA 触发词 + 实体蓝图词 + 技法抽屉词 + 反向防错词` 组装成真正参与生成的极简 Tags | 是 |

> 记忆口诀：**菜单 2.5 的训练测试词 ≠ 换轨口令 ≠ 菜单 3 的资产需求口令 ≠ 最终生图 Prompt**。

## 现在二者到底怎么衔接

菜单 2.5 的任务，是把你在探索期摸出来的**整体画风**固化到 LoRA 里。等 LoRA 训练完成后，菜单 2.5 的测试词就完成历史使命了，不再进入量产阶段。真正带着画风进入菜单 3 的，不是那串测试词本身，而是这三个东西：**LoRA 文件名、LoRA 触发词、双轨分发规则**。

进入菜单 3 后，你不再自己手写长 Prompt。你只要提资产需求，我来把需求拆到四个物理层：`Load LoRA` 负责挂上风格；`Text Prompt` 负责实体蓝图词与技法抽屉词的极简组合；`ControlNet` 负责姿态/透视/结构约束；`KSampler` 负责采样安全区参数。这样就不会再出现“菜单 2.5 那串词要不要继续粘到菜单 3 里”的混乱。

| 菜单阶段 | 真正继承下来的东西 | 被废弃的东西 |
|---|---|---|
| **菜单 2 → 2.5** | 参考图探索出的目标画风 | 每次都依赖参考图重新算嵌入的临时状态 |
| **菜单 2.5 → 3** | `LoRA 文件名 + 触发词 + 双轨规则` | Civitai 训练页里的测试预览词、IPAdapter 垫图逻辑 |
| **菜单 3 量产** | `实体蓝图 × 技法抽屉` 的节点化分发 | 用户自己手写长 Prompt |

## 我已经替你优化好的两段可复制模板

### 1. 启动换轨口令

```md
LoRA 已就绪，文件名为 `trickster_style_e10.safetensors`，触发词为 `trickster_style`。请先把它登记为菜单3当前默认量产 LoRA，并按双轨架构给我下发菜单3的【主角跑动测试工单】。注意：我后续只提资产需求，不再手写最终生图 Prompt；你负责把最终 Prompt 拆入 Load LoRA / Text Prompt / ControlNet / KSampler 四区。
```

### 2. 菜单 3 资产需求口令

```md
Manus，我要做游戏资产：**【👉 需求：例如，[交互物] 木制宝箱开启的 3 帧动画 / 或 [陷阱] 静态地刺】**。
这段话只是**资产需求口令**，不要把我的整段中文需求原样塞进最终生图 Prompt。
请启动『十字双轨组装』流程，查阅 `prompts/PROMPT_RECIPES.md`：

0. **接棒确认**：先告诉我你当前量产底座使用的是哪个 `Base Model`、哪个 `LoRA 文件名`、哪个 `LoRA 触发词`。
1. **双轨抓药声明**：明确告诉我你本次提取了下半部的哪个【实体蓝图】（长什么样）？并且拉开了上半部的哪几个【知识抽屉】（怎么画）？(注：比如做静态陷阱严禁混入动画规则，遇冲突执行熔断)。
2. **输出「节点化」制造工单**：请给我一份可直接在 ComfyUI 本地照抄的 Markdown 工单。严格按以下节点分类输出：
   - 🖼️ **[Load LoRA 节点]**：项目专属 LoRA 的名称、触发词及推荐权重。
   - 📝 **[Text Prompt 节点]**：只写最终会实际喂给模型的极简 Tags（`LoRA触发词 + 实体蓝图词 + 技法抽屉词 + 反向防错词`），要求 <20 词；**禁止**把我的整段需求句原样复制进去。
   - 🤖 **[ControlNet 节点]**：根据 🏃[动画] 或 📐[透视] 抽屉的约束，明确告诉我该启用哪个模型（OpenPose/Canny/Depth），以及我该准备什么样的参考底图喂给它防崩坏。
   - ⚙️ **[KSampler 节点]**：根据 ⚙️[AI参数] 抽屉，给出建议的 CFG 和 Denoising 安全数值。
绝对不要在对话框写小作文，只给我可以直接 Copy 的节点图纸！
```
