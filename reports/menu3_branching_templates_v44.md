# 菜单 3 分支化工单体系（V4.4）

本次升级采用了当前项目里**效率与质量兼顾的最优方案**：不再让一个万能需求口令同时承担静态资产、动画序列和透视场景三种任务，而是先按结构风险分流，再组装节点。这样做的核心收益，是把返工率最高的三类错误——**静态物误混动画词、动画帧骨折漂移、场景透视塌陷**——在入口处就提前隔离。

| 分支 | 你什么时候用 | 核心保护目标 | 默认结构策略 |
|---|---|---|---|
| **静态资产版** | 单个陷阱、单个宝箱、单个地形块、单帧交互物 | 轮廓稳定、材质不串味、方便批量量产 | 非生物优先 `Lineart + Canny`，简单单体可空结构约束 |
| **动画序列版** | 跑步、跳跃、攻击、巡逻、开合等多帧任务 | 防滑步、抗骨折、帧间一致性 | 生物强制 `DWPose/OpenPose`；运动件视情况用 `Lineart + Canny`；必要时走 `Img2Img` |
| **透视场景版** | 关卡段、房间、假游戏截图、复杂机关区域 | 地平线、平台厚度、遮挡关系、可落脚空间正确 | `Depth` 优先，必要时叠 `Canny` 锁边 |

> **一条总原则**：如果这次任务最怕的是“动起来会坏”，优先走**动画序列版**；如果最怕的是“空间会塌”，优先走**透视场景版**；如果只是做可复用素材，优先走**静态资产版**。

## 直接复制模板

### 静态资产版

```md
Manus，我要做 **菜单3-静态资产版** 游戏资产：**【👉 需求：填写具体静态资产，例如 [陷阱] 静态地刺 / [交互物] 关闭状态宝箱 / [地形] 断裂砖块】**。
请按『静态资产分流』处理，不要误拉动画抽屉。
先执行接棒确认，然后从 `prompts/PROMPT_RECIPES.md` 中抽取最匹配的【实体蓝图】与【光影/参数】抽屉；如果资产涉及明确空间朝向，再补【透视】抽屉。
请输出可直接复制的 ComfyUI 工单，并严格分成四区：
- 🖼️ **[Load LoRA 节点]**：LoRA 文件名、触发词、权重。
- 📝 **[Text Prompt 节点]**：只写 `LoRA触发词 + 实体蓝图词 + 光影抽屉词 + 反向防错词`；若无必要，禁止混入动画词。
- 🤖 **[ControlNet 节点]**：默认按非生物硬表面处理；告诉我是否启用 `Lineart / Canny`，以及底图需要是纯轮廓图、线稿图还是可空。
- ⚙️ **[KSampler 节点]**：给出最稳的静态资产参数，不要为了速度牺牲边缘稳定性。
只给节点图纸，不写解释。
```

### 动画序列版

```md
Manus，我要做 **菜单3-动画序列版** 游戏资产：**【👉 需求：填写具体动画，例如 [角色] 主角跑动 8 帧 / [角色] 跳跃三阶段 / [交互物] 宝箱开启 3 帧】**。
请按『动画序列分流』处理，强制优先遵守 **物理防滑步 > 空间透视正确 > 光影材质表现**，严禁把静态单帧策略直接套到序列帧。
先执行接棒确认，然后从 `prompts/PROMPT_RECIPES.md` 中抽取最匹配的【实体蓝图】与【动画/解剖/参数】抽屉；若镜头有明显角度变化，再补【透视】抽屉。
请输出可直接复制的 ComfyUI 工单，并严格分成四区：
- 🖼️ **[Load LoRA 节点]**：LoRA 文件名、触发词、权重。
- 📝 **[Text Prompt 节点]**：只写 `LoRA触发词 + 实体蓝图词 + 动画抽屉词 + 解剖抽屉词 + 反向防错词`，要求极简。
- 🤖 **[ControlNet 节点]**：如果是生物关节角色，强制指定 `DWPose/OpenPose`；如果是机械或硬表面运动件，说明是否改用 `Lineart + Canny`；同时告诉我每一帧参考底图该如何准备，避免骨折与漂移。
- ⚙️ **[KSampler 节点]**：给出最稳的序列参数；若使用 Img2Img，请明确 `Denoising Strength` 安全区间，优先保证帧间一致性而不是单帧花哨。
只给节点图纸，不写解释。
```

### 透视场景版

```md
Manus，我要做 **菜单3-透视场景版** 游戏资产：**【👉 需求：填写具体场景，例如 [地形] 地下城平台区透视场景 / [场景] 带前景立柱的机关房间 / [关卡] 可横版游玩的假截图】**。
请按『透视场景分流』处理，优先解决地面水平线、平台厚度、前后景遮挡和可落脚空间正确性，不要误用角色动画抽屉。
先执行接棒确认，然后从 `prompts/PROMPT_RECIPES.md` 中抽取最匹配的【实体蓝图】与【透视/光影/参数】抽屉；只有场景内存在角色动作约束时，才额外补【动画/解剖】抽屉。
请输出可直接复制的 ComfyUI 工单，并严格分成四区：
- 🖼️ **[Load LoRA 节点]**：LoRA 文件名、触发词、权重。
- 📝 **[Text Prompt 节点]**：只写 `LoRA触发词 + 实体蓝图词 + 透视抽屉词 + 光影抽屉词 + 反向防错词`，禁止把整段场景描述原样塞进去。
- 🤖 **[ControlNet 节点]**：优先说明是否启用 `Depth`，是否需要叠加 `Canny` 锁硬边结构，并明确我要准备哪种场景草图或深度底图。
- ⚙️ **[KSampler 节点]**：给出最稳的场景参数，优先保证空间结构清晰与边缘可切图性。
只给节点图纸，不写解释。
```

## 最后怎么选

| 任务长相 | 直接选项 | 原因 |
|---|---|---|
| 单个陷阱、单个砖块、单个宝箱、单帧物件 | **静态资产版** | 追求稳定量产，没必要加动画负担 |
| 主角动作、怪物动作、开合动作、多帧循环 | **动画序列版** | 首要矛盾是帧间一致性 |
| 房间、平台区、复杂机关场景、假截图 | **透视场景版** | 首要矛盾是空间结构 |
| 角色动作 + 强透视场景同时存在 | **先动画序列版，再透视补强工单** | 先保运动，再保空间，返工成本最低 |
